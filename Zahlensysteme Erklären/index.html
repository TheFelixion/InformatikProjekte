Binärsystem

Das Binärsystem, auch Dualsystem genannt, ist ein Stellenwertsystem zur Basiszahl 2.
Es werden nur die Ziffern 0 und 1 genutzt. Die Ziffern werden von rechts nach links mit der 0 beginnend durchnummeriert.
Der Wert der Ziffer im Dezimalsystem ergibt sich aus dem Produkt der Ziffer mit der Potenz der Basiszahl 2
und der Nummer der Ziffer als Exponent:

Hexadezimalsystem

Das Hexadezimalsystem, auch Sechzehnersystem genannt, ist ein Stellenwertsystem zur Basiszahl 16.
Da es nur einstellige Zahlen von 0-9 gibt, werden zusätzlich die Buchstaben A-F
als „Nichtstandardziffern“ nach dem folgenden System verwendet:

Ziffer0123456789ABCDEF
Wert0123456789101112131415

Die Ziffern werden hintereinandergeschrieben. Dabei ist jede Ziffer ihr Wert
multipliziert mit der 16er-Potenz mit der Hochzahl der Nummer Ziffer, begonnen von rechts mit 0:

Farben und das Hexadezimalsystem

Das Bildschirme Farben mit Rot, Grün und Blau darstellen, sollte ja bekannt sein.
Meist werden hierfür 8 Bit, also 1 Byte pro Farbe benutzt. Das heißt, dass man z.B.
der Farbe Rot einen beliebigen Wert zwischen 0 und 255 zuweisen kann oder eben eine
beliebige Hexadezimalzahl zwischen 00 und FF.
Dies kann man z.B. in der RGB-Farbdarstellung darstellen: rgb(255,0,0) für Rot.
Daraus folgt der bekannte Hex-Farbcode zur Farbdarstellung.
Sie beginnt mit einem Hashtag(#), gefolgt von zwei Hexadezimalen für Rot, zwei für Grün und zwei für Blau.

Einerkomplement

Bisher können nur positive Zahlen mit dem Binärsystem dargestellt werden, um nun auch negative Zahlen
darstellen zu können, gibt es das Einerkomplement. Das erste Bit stellt nun keinen Teil der Zahl mehr dar,
sondern ist ein Indikator dafür, ob die Zahl positiv oder negativ ist.
Hierbei steht eine 0 für positiv und eine 1 für negativ.
Um eine negative Zahl im Einerkomplement darzustellen, wird zuerst der Betrag der Zahl als Binärzahl dargestellt.
Bei dieser Binärzahl werden nun alle einzelnen Bits invertiert, das heißt aus jeder 0 wird eine 1 und umgekehrt.
Das Invertieren sorgt dafür, dass Addition, sowie Subtraktion weiterhin funktionieren.

Beispiele -5 → |-5| = 5 = 01012 → 10102 -3 → |-3| = 3 = 00112 → 11002 Probleme

Im Einerkomplement gibt es zwei Darstellungen für die 0: 0000 und 1111.
Das stellt zwar kein mathematisches Problem dar, da -0 = 0 ist, aber es ist ineffizient.
Ein größeres Problem ist, dass die Addition über die 0 hinweg nicht funktioniert:

Wertebereich

von: -2n-1 + 1
bis: 2n-1 -1

Zweierkomplement

Eine weitere Darstellung für negative Zahlen ist das Zweierkomplement. Mit dem Zweierkomplement
werden die zwei Probleme des Einerkomplements gelöst. Aus diesem Grund wird standartmäßig das Zweierkomplement verwendet.
Das Zweierkomplement bildet man, indem man zuerst das Einerkomplement bildet und dann eine binäre 1 addiert.

Beispiele -5 → |-5| = 5 = 01012 → 10102 (Einerkomplement) → 10102 + 00012 = 1011 -3 → |-3| = 3 = 00112 → 11002 (Einerkomplement) → 11002 + 00012 = 1101 Beispiel, dass die Addition über 0 nun funktioniert: Wertebereich

von: -2n-1
bis: 2n-1 -1
